# MVCモデルって何や？
MVCとは3つの役割に分けてアプリケーションを作る方法の事で、それぞれ違う仕事を担当している。

Modal(モデル)
→ データの管理係。dbとやり取りしたり、計算したりする。 

View(ビュー)
→ 見た目担当。ユーザーに見せる画面やHTMLを作る。

Controller(コントローラー)
→ 指令係。ユーザーの操作を受け取って、ModelやViewに指示を出す。


ラーメン屋で例えると...
ラーメン屋さんをアプリに見立ててみる。
・お客さんが「味玉ラーメンください！」という→ Controllerが注文を受ける

・Controllerが「味玉ラーメン作って！」とModelに指示を送る。→ Modelがレシピ通りに作る。

・出来上がったラーメンをViewがきれいに盛り付けてお客さんに見せる

つまり、Controllerが「橋渡し役」になって、ModelとViewを繋げている！


## Railsでの実際の動き
例えば「ユーザー一覧を表示する」という機能があるとする。

①ユーザーが「一覧を見たい」とURLにアクセスする。

②Controllerがそのリクエストを受け取る(例：UsersControllerのindexアクション)

③Model(例：User)がデータベースからユーザー情報を取ってくる

④View（例：index.html.erb）がその情報をHTMLにして表示する。


# まとめ
・MVCは「分業制」のようなものであり、役割を分けることによってコードがすっきりする。

・Railsはこの仕組みをうまく使って、初心者でも作りやすいようになっている。
